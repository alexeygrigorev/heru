## Public API surface — do not break without a semver bump

Anything **not** prefixed with `_` is public. Changing the signature,
return type, or behavior of any of the following is a **semver-major**
breaking change that requires a version bump in `pyproject.toml`, a
CHANGELOG entry, and a migration note for litehive:

- `heru.get_engine(name)`, `heru.ENGINE_CHOICES`
- `heru.base.ExternalCLIAdapter` (subclass, don't modify shape)
- `heru.base.CLIInvocation`, `heru.base.CLIExecutionResult`
- Each adapter class: `CodexCLIAdapter`, `ClaudeCLIAdapter`,
  `CopilotCLIAdapter`, `GeminiCLIAdapter`, `OpencodeCLIAdapter`,
  `GozCLIAdapter`
- Every pydantic model in `heru/types.py`
- `heru.main.main` (the CLI entry)

**Internal** (change freely, no semver implications): every
`_`-prefixed module, every private method on public classes, and the
exact text of rendered transcripts.

3. **Stage reports are a litehive concept, not heru's.** `StageReport`,
   `StageResultSubmission`, `StageResultTests`, and `TaskUpdateSubmission`

   or fields inside heru. If your task requires you to edit stage

4. **Tests must be standalone.** No test under `tests/` may import

   hand-craft strings or files under `tests/fixtures/` — do NOT

5. **Respect the contract tests** (once `tests/contract/` exists per
   T-0007). Changing an assertion in `tests/contract/` is by

   checklist.

6. **Every new adapter needs both a public class AND a `_impl` helper
   module.** Shape: `adapters/<engine>.py` for the public class,
   `adapters/_<engine>_impl.py` for parsing/quota internals. Keep the

Goal:
Today each heru adapter (codex, claude, copilot, gemini, opencode, goz) streams its engine's native JSONL payloads directly to stdout. Callers have to parse 6 different event schemas to extract agent messages, tool calls, tool outputs, errors, and usage. Define a single unified event envelope that heru emits regardless of the underlying engine. Each adapter translates its native events into the unified shape before streaming. The native payload stays embedded as a 'raw' field for consumers that need engine-specific detail.

Acceptance criteria:
- Define a UnifiedEvent schema (pydantic model) with fields like: kind (message|tool_call|tool_result|usage|error|status|continuation), engine, sequence, timestamp, role, content, tool_name, tool_input, tool_output, error, usage_delta, continuation_id, raw (the original native payload)
- Each adapter implements a translate_native_event(native_payload) -> UnifiedEvent | None method that maps engine-specific events to the unified shape
- ExternalCLIAdapter.run gains an emit_unified mode that streams UnifiedEvent JSONL to stdout instead of native payloads
- heru CLI defaults to emit_unified output; a --raw flag opts back into native passthrough for debugging
- Unit tests per adapter with fixture-based native event samples asserting the translated UnifiedEvent shape
- Docs section in README.md describing the unified event schema
